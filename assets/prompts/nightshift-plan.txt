# PLANNING TURN

The task queue is low. Do not write code or open a PR this turn. Queue depth is not success; a small set of independent, verifiable tasks is.

1. Read `.nightshift/followups.md` when present and the project's `objective.md` from the devbrain brain. Inspect `devbrain-todo list all` so open, taken, review, held, and recently completed work are not duplicated.
2. Identify the next objective gap, then inspect the live repository enough to localize candidate work to likely files, symbols, or tests.
3. Add 1 to 3 tasks only after every candidate passes the quality gate below. If no candidate passes, do not invent work; record the concrete missing decision in `.nightshift/followups.md` and end the turn.

## TASK QUALITY GATE

- One outcome: one observable end state that fits one reviewable PR.
- Localized: name likely files, symbols, tests, or an exact discovery command.
- Scoped: state what is in scope and at least one non-goal.
- Verifiable: include deterministic acceptance evidence and an executable validation command.
- Runnable: name dependencies; enqueue only work whose dependencies are complete or explicitly ordered.
- Parallel-safe: assign a conflict key (the smallest shared file/package/domain). Do not enqueue parallel tasks with overlapping conflict keys.
- Bounded: target one worker turn. Split larger work into dependency-ordered vertical slices.

Reject vague tasks such as "audit", "improve", "cleanup", "research", or "refactor" unless they name a concrete artifact, scope, and acceptance check.

## TASK BODY CONTRACT

Use `devbrain-todo add "title" -p PRIORITY -b "..."` and give every task body these exact labels:

Outcome: <single observable result>
Evidence: <repo location, failing test, prior decision, or discovery command>
Scope: <allowed files/packages/behavior>
Non-goals: <explicitly deferred work>
Acceptance: <deterministic proof the outcome is complete>
Verify: <command the worker can run>
Depends on: <task id(s) or none>
Conflict key: <smallest shared path/package/domain>
Budget: one worker turn

Deduplicate against every active state, enqueue the smallest useful batch, then end the turn.
