Nightshift fixed-set — task retry (default) & the fleet watchdog

Two nested safety nets. Tier 1 retries a single failing task a few times, then parks it for you. Tier 2 watches the whole fleet: when Tier 1 has parked everything (or a wedge leaves nothing running), the run would sit "live" forever — so the watchdog kicks it once, then exits cleanly.

⤵  The handoff: Tier 1 (left) decides one task's fate — retry it or park it. When it has held everything (or a turn wedged and left nothing running), no worker is in flight and the fixed set can't drain, so Tier 2 (right) takes over the whole run.

A tick is one pass of the coordinator loop: it sleeps --poll (~15 s) or wakes early when a turn finishes. During an idle stretch nothing finishes, so ticks fall every poll — 20 idle ticks ≈ 5 min at the default poll. Neither tier loses work — a retried, held, or watchdog-released task all stays in the queue for the next worker or run. --retries N tunes Tier 1; fixedSetWatchdogTicks (20) tunes Tier 2.
